# 「考试题型」例题重构规范（实例参考 · 所有更新 agent 必读）

> **这是一份实例参考**（某金融计量课程沉淀），演示的方法是通用的：期中/期末前拿到「考试只考哪几类题型」的新信息后，如何给并行更新 agent 写一份规范，把已有讲义的例题批量重构成与考试题型对齐的形式。给你的课程写同类规范时，替换其中的题型约束与学科示例即可。核心可复用点：① 只改例题、其余正文逐字保留的**最小改动原则**；② 「读者是否需要拿笔/计算器算出一个数」这条**计算题判定标准**；③ 末尾的渲染硬规则与单行回报协议。

## 新信息：考试题型

期末考试**只有两类题：证明题 + 简答题**。**没有选择题、填空题、判断题、计算题。禁止携带计算器。**

含义：考试不会出"代入具体数字算出一个数值答案"的题。所以讲义里的**例题**必须据此重写——把"计算题"改成"证明/推导题"或"简答题"。

## 你要做什么

**只改两类东西，其余正文一律原样保留：**

1. **「典型例题」小节**：把每道例题改写成**证明题**或**简答题**：
   - **证明题**：要求**符号推导 / 证明**一个结论，**不代入具体数字**。例：证明 GARCH(1,1) 无条件方差 $\bar\sigma^2=\omega/(1-\alpha-\beta)$；证明 ARCH(1) 峰度 $=3(1-\alpha_1^2)/(1-3\alpha_1^2)>3$；推导 GARCH 多步预测向 $\bar\sigma^2$ 收敛；从均值-方差一阶条件推出 CAPM 关系；证明随机游走下 $VR(2)=1$；证明正态同方差下 MLE=OLS。
   - **简答题**：要求**解释 / 比较 / 说明 / 写出步骤或框架**，讲逻辑、特点、结论。例：简述 FHS 算法步骤；解释 News Impact Curve 如何比较 GARCH/GJR/EGARCH；比较 RW1/RW2/RW3；说明联合假设问题；简述 DCC 两步估计；说明边际分布为何影响 VaR；写出某检验的四段（原假设→统计量→分布→拒绝规则）。
   - 每道例题保留"解 / 解答 / 解题示范"，但**解答里也不要出现需要计算器的多步算术**——用符号推导、定性说明、写出公式与判据即可（如检验题写出统计量与临界值的**比较逻辑**，但不要求算出统计量的具体数值）。

2. **内嵌的 `> [!example]` 数值例子**：
   - 如果它是"给一组数字、要求一步步算出数值结果"的**计算演练**（如"给 $\omega=0.2,\alpha=0.5$ 算无条件方差/峰度/2 步预测""给 $T=500,x=38$ 算 $LR$""给 $\hat\sigma=1.5\%$ 算 VaR 多少元""给 $Q_t$ 矩阵求 $R_t$"）→ **改成符号推导版（证明题风格）或简答版**，删去具体数字代入与算术。
   - 如果它只是用一两个数字**点一下直觉**（如"$\alpha+\beta=0.95$ 说明持续性很高""正态 1% 分位 $-2.326$ 比经验的 $-2.656$ 浅，故低估极端损失"这种**一句话定性对比、无需动笔算**的），可以**保留**——这不是计算题，是帮助理解的说明。判断标准：**读者是否需要拿笔/计算器算出一个数**。需要算 → 改写；只是看一眼数字理解结论 → 保留。

## 改写示例（必看）

- 计算题（删）："估得 $\hat\sigma_{t+1}=1.5\%$，求 99% VaR；组合市值 200 万，VaR 多少元？"
- 改成证明/简答："写出基于 GARCH 的 99% 条件 VaR 表达式 $VaR_{t+1}=-\hat\mu-\hat\sigma_{t+1}F^{-1}_\alpha$，**推导**它来自'让 $\Pr(r_{t+1}<-VaR)=\alpha$'，解释为什么用 $F^{-1}_{0.01}=-2.326$（负号、左尾），并说明组合 VaR 如何由收益率 VaR 乘市值换算（只写换算关系，不要求算出具体金额）。"

- 计算题（删）："$T=500$ 天违例 $x=38$ 次，算 Kupiec $LR$ 并判断。"
- 改成简答："写出 Kupiec 无条件覆盖检验的四段（$H_0$ / 似然比统计量 / $\chi^2(1)$ 分布 / 与 3.84 比较的拒绝规则），说明它检验什么、它的局限（只看违例**次数**、不看违例是否**聚集**），并说明 Christoffersen 检验如何补足。"

## 渲染硬规则（务必遵守，违反会导致 PDF 缺字/编译失败）

- 所有数学符号用 LaTeX `$...$` / `$$...$$`，**严禁正文直接打 Unicode 希腊字母或数学符号**（α β σ ⇒ ≈ ≤ ≥ 等写成 `$\alpha$`、`$\Rightarrow$`）。正文 `→`、`、`、`×` 可保留。
- **闭合的 `$` 后不能紧跟数字**：写 `$\approx 0.9$` 不写 `$\approx$0.9`；数字写进数学段内。
- 数学环境内的中文用 `\text{...}` 包裹。
- **不要用 ``` 代码块**；步骤用编号文字清单。callout 用 `> [!note]/[!tip]/[!warning]/[!example] 标题`（标签+标题同行、正文换行）。
- 不用 `\usepackage`、`\newcommand`、`enumitem`、mermaid、HTML、圈数字。

## 输出

把更新后的整讲写入指定输出路径（**与输入同一路径，原地更新**），然后只回复一行：本讲改了哪几道例题（从计算题→证明/简答）、是否动了 inline 数值例子。**除指定要改的例题/数值演练外，其余正文逐字保留。**
