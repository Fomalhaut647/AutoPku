# 讲义「考纲对齐」更新规范（实例参考 · 所有更新 agent 必读）

> **这是一份实例参考**（某金融计量课程沉淀），演示的方法是通用的：老师给出明确考纲后，如何给并行更新 agent 写一份规范，对已有讲义做面向考纲的瘦身重构。给你的课程写同类规范时，替换考纲清单与学科示例即可。核心可复用点：① 「定量掌握 / 定性掌握 / 考纲外」三级定位与统一标记写法（§四）；② 考纲外内容「删具体章节推导、留被别处依赖的概念精简版」的删减尺度（§二末段）；③ 渲染硬规则与单行回报协议。

这是对一份已有的、内容齐全的零基础速通讲义做**面向期末考纲的瘦身重构**。老师已给出明确考纲，本次更新有三条总原则：

1. **按考纲重新定位**：考纲把内容分成「定量掌握」（要会写公式/推导/估计步骤/算法思路）和「定性掌握」（会解释/比较/说结论）。讲义要让读者一眼看出每个知识点属于哪类、要掌握到什么程度。
2. **精简删减考纲外内容**：考纲未列入的内容（见下）大幅删减，讲义聚焦考点。
3. **编程题只考算法思路、不考手写代码**：所有「估计/编程」内容只保留**算法思路 / 估计步骤**（伪代码或编号步骤清单级别），**删除全部 MATLAB 语法代码**，也不必讲 MATLAB 语法。

## 一、官方考纲（必须内化）

### 定量掌握（要能写出公式 / 推导 / 估计步骤 / 算法思路）

1. Random Walk 模型检验（重点 RW1，最简单情形下如何检验收益率可预测性）
2. ARCH 和 GARCH 基准模型的简单推导（设定、GARCH(1,1)、无条件方差推导、参数含义如波动持续性 α+β）
3. GJR-GARCH、EGARCH、GARCH-in-Mean 模型（形式 + 各自解决什么问题）
4. **GARCH 模型的 MLE 估计（正态分布假设下）与编程实现**（老师录音特别强调：写出 log-likelihood、估计思路、实现步骤）
5. GARCH(1,1) 的 1 步和多步预测（多步会回归到长期方差）
6. 预测比较：M-Z（Mincer-Zarnowitz）回归、DM（Diebold-Mariano）检验
7. Realized GARCH(1,1) 模型与估计
8. DCC 模型（模型设定 + 两步估计流程）
9. 基于 GARCH 模型的 VaR 预测
10. **Filtered Historical Simulation（FHS）**（老师录音特别点到，要掌握算法步骤）
11. Mean-Variance 模型（问题设定 + 一阶条件）
12. 从 MV 到 CAPM 的证明（**基于组合构建的证明**，重在思路）
13. CAPM、三因子模型的形式（也知道可扩展到四/五/多因子）

### 定性掌握（会解释 / 比较 / 说结论，不抠字眼）

1. 有效市场假说与收益率可预测性（含主要实证结论）
2. 有效市场假说与投资（技术分析 / 基本面分析 / 量化投资）
3. Random Walk 的 RW1/RW2/RW3 区别含义（强弱：iid → 独立不同分布 → 不相关）
4. Variance Ratio 检验的**基本思路**（不要求重推导渐近方差）
5. Joint Hypothesis Test 批评
6. 波动率 / 风险 / 不确定性的区别
7. GARCH 模型的各种扩展思路（为什么要扩展）
8. News Impact Curve 的理解与比较（GARCH/GJR/EGARCH 的非对称性）
9. GARCH 中边际分布的选择（为什么不能只用正态：厚尾/偏度）
10. 高频金融数据（RV）的特征（简单掌握：日内信息、微观结构噪声）
11. Realized GARCH 模型的特点
12. VaR 预测中边际分布的影响（尾部风险、正态低估极端损失）
13. VaR 预测评估（绝对：失败率 / Kupiec / Christoffersen；相对：损失函数比较）
14. Mean-Variance 图形分析、主要结论（可行集、有效前沿、最小方差组合、无风险资产、资本市场线、最优风险组合）
15. 从 CAPM 到三/四/五/多因子模型的理解（发展逻辑、alpha 与 beta 含义）
16. CAPM 和多因子模型的计量检验框架（时间序列回归 / 面板回归 / Fama-MacBeth；核心看 α=0；不用特别细）

### 老师答题提醒（写进讲义答题策略）

- 内容一定来自课件/课堂；定量部分尽量写公式/推导/估计步骤；不会推导也要写出对问题的理解；定性题能讲逻辑/特点/结论就有分；开放题多写一点。

## 二、考纲外内容（本次大幅删减）

考纲**完全未提及**，删减或压成极短背景：

- **时间序列的 ARMA 估计、ACF/PACF 定阶、ADF/单位根检验**（仅保留 GARCH 需要的「平稳性、白噪声、ARCH 即 ε² 的 AR」这点直觉作背景）。
- **三大检验 Wald/LM/LR 的详细推导、White 异方差检验、三明治协方差推导、Bootstrap、遗漏/多余变量、LAD、Cramér-Rao**（第 1 讲只保留 OLS/回归速记 + MLE 思想，因为 GARCH-MLE 和因子回归要用）。
- **基本面量化投资全部**（估值因子、质量因子、应计异象 Sloan、RNOA、PEAD、剩余收益模型等，原第 8 讲）——考纲未列，压成一页「了解即可」的简表或直接标注考纲外。
- **Variance Ratio 检验的渐近方差重推导（V_N/V_O、Z 统计量完整推导）**——只留「基本思路」（长短期方差成比例）。

注意区分：被删的是上述**具体推导/章节**，不是相关概念的一切痕迹。如「平稳性」「OLS 回归形式」「MLE」这些被别处用到的，保留精简版。

## 三、Markdown / 数学 / 排版硬规则（违反会导致 PDF 渲染缺字或编译失败，务必遵守）

最终用 `pandoc + XeLaTeX + ctex` 渲染。**上一版踩过大量坑，这次务必从源头避免：**

- **数学一律用 LaTeX**：行内 `$...$`、独立 `$$...$$`，所有符号用反斜杠命令（`\Rightarrow \approx \le \ge \neq \alpha \beta \sigma \sum \int \partial` 等）。
- **严禁在正文/标题里直接打 Unicode 数学符号或希腊字母**（⇒ ≈ ≤ ≥ ≠ ⊂ ⊗ ∂ α β γ σ ε θ μ ω 等）——一律写成 `$\Rightarrow$`、`$\alpha$`。正文里的 `→`、`、`、`×`、`·` 这类可保留（能渲染）。
- **关键陷阱：闭合的 `$` 后面不能紧跟数字**（pandoc 会解析失败把后文吞进数学）。所以**把数字写进数学段内**：写 `$\approx 0.91$` 不要写 `$\approx$0.91`；写 `$\alpha+\beta<1$` 不要写 `$\alpha+\beta$<1`。
- **数学环境内若必须出现中文，用 `\text{...}` 包裹**：如 `$CoVaR^{\text{困境}}$`、`$\Pr(r\le q_\alpha)=\alpha\ (\text{即左尾})$`。
- 期望/方差/相关写 `\mathrm{E}`、`\mathrm{V}`、`\mathrm{Cov}`、`\mathrm{Corr}`。重要结论用 `\boxed{...}`。
- **算法思路 / 估计步骤一律用「编号文字清单」呈现**（中文步骤，渲染在正文字体，安全），**不要用 ``` 代码块**，更不要写 MATLAB 语法。例：
  > **GARCH(1,1) 正态 MLE 估计步骤**
  > 1. 给定参数 $(\omega,\alpha,\beta)$ 与初值 $\sigma_1^2$（取样本方差），按 $\sigma_t^2=\omega+\alpha\epsilon_{t-1}^2+\beta\sigma_{t-1}^2$ 递推出全部 $\sigma_t^2$。
  > 2. 代入对数似然 $\ell=\sum_t[-\tfrac12\log(2\pi)-\tfrac12\log\sigma_t^2-\tfrac{\epsilon_t^2}{2\sigma_t^2}]$。
  > 3. 用数值优化器在约束 $\omega>0,\alpha\ge0,\beta\ge0,\alpha+\beta<1$ 下最大化 $\ell$（等价最小化 $-\ell$）。
  > 4. 用对数似然在最优点的 Hessian（信息矩阵）求参数标准误。
- **callout 框**（克制用，每小节 1–2 个）：`> [!note] 直觉`、`> [!example] 例`、`> [!warning] 易错`、`> [!tip] 技巧`，标签与标题同一行、正文换行。
- 表格用标准 Markdown 管道表格。列表用 `1./2./3.` 或 `-`，**不要圈数字、不要希腊字母当列表标签**。
- **禁止**：`\usepackage`、`\newcommand`、`enumitem`、`mermaid`、HTML 标签、`\tag{}`。

## 四、考纲定位标记（统一写法）

- 每讲 H1 标题下用一行引用块标注本讲考纲定位，例如：
  `> 考纲定位：本讲属**定量核心**（GARCH 基准推导、MLE 估计、预测——多个定量考点）。`
  或 `> 考纲定位：本讲**考纲外**（仅作背景，时间紧可跳过）。`
- 讲内每个对应考点的小节，在标题或开头用 `**[定量]**` / `**[定性]**` / `**[考纲外]**` 标签点明掌握程度。

## 五、输出

把更新后的整讲 Markdown **写入指定输出文件路径**，然后**只回复一行**：标题 + 大致字数 + 本讲做了哪些改动（删了什么、加固了什么、标注了什么）。直接输出可进 PDF 的正文，不要元叙述。
