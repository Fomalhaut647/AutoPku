# 「速通讲义」共享写作规范（模板 · 所有写作 agent 必读）

> 来自 autopku speed-notes-kit。拷到 `<课程>/<讲义名>/.build/STYLE_SPEC.md` 后：
> 通用部分（§0 §2 §3 §5 §7 §8）一般不动；**§1 参考资料、§4 记号约定、§6 考情分析按本课填**。
> 区分两种产物：用户说「复习/我学过」→ 简明提纲（定义+套路+范例+易错点）；
> 说「速通学习/没学过/零基础」→ 下面这套教材式讲义。不确定就问用户。

## 0. 一句话任务与读者画像
把本课某一章，写成「长期没接触、几乎零基础的读者也能读懂、并据此应付期末」的教学讲义章节。
读者：学过一点但忘得差不多、时间有限、目标是会做本课期末题。所以既要详细到能教会（不假设记得任何公式），又要精炼到能快读完（不堆废话、不写考不到的高阶内容）。
**核心心法：以往年真题为纲（§6）。真题考过的题型/公式/套路讲深讲透配真题级例题；真题没碰过的边角内容一律不写或一句带过。**

## 1.〔按课程填〕参考资料与优先级
**优先级：往年题（最高）> 作业题+参考答案（次）> 课件（再次）。** 目录里**可能没有往年题**——没有就不管往年题，退而用作业+课件。列出本课具体文件位置；忽略学生本人作答 PDF；课件用 `pdftotext -layout` 或预提取的 `.build/sources/*.txt` 读。

## 1b. 内容范围：期中 vs 期末（主控 agent 派发前先定，写进各章任务指令）
- **期中讲义** → 只覆盖前半学期，**不要**加入后半学期内容。
- **期末讲义**：① 本课**只有期末没期中** → 覆盖**整学期**；② **既有期中又有期末** → **重点输出后半学期**，前半学期只做**简略复习**（不重复详写已考过的）。
- **判断知识点属前半/后半学期**：若有**期中往年题**就用它界定（期中考过的≈前半学期）。

## 2. 深度档位：教材式零基础
每个知识点按六步展开（按考试权重定详略，高频写满低频压缩）：
1. **直觉与动机**：1–3 句白话讲「要解决什么问题、为什么这么定义」，配现实小情境。
2. **严格定义并逐符号解释**（零基础读者不认识记号）。
3. **求解方法**：把这类题的标准解法拆成可照做的步骤，每步讲清理由，不只丢公式。
4. **完整例题**：至少 1 个、高频考点 2 个，算到底每步都写，优先真题/作业改编。
5. **易错点**：用 `> [!warning] 易错`。
6. **真题精练**：挑 1–2 道代表真题给完整解答（不要把所有真题全解，全解另出单文档）。

## 3. 文档结构模板（每章一个 md 文件）
```
# 第 N 章　中文标题            <- 首行就是 H1，将成为 PDF 一章；不要写 YAML
> 一句话本章地图。
> 考情提示：本章对应往年哪些题型、占多少分量。
## N.0 前置速补（可选）
## N.1 <主题>  ... ## N.2 <主题>   <- 章内用 ## / ###
## N.x 真题精练
## 速记卡（考前最后过一遍）      <- 浓缩必背公式/结论/套路
```
- **章节编号手写进标题**（如 `## 5.3 边缘分布`），自动编号已全局关（`secnumdepth=-1`），否则双重编号。
- 文件按 `00-,01-,…,NN-` 数字前缀命名控制拼接顺序；`00-` 通常是导读（可由主控 agent 亲自写，含考情分析）。

## 4.〔按课程填〕记号约定
列出全讲义统一记号，避免各写作 agent 各写一套。示例（概率统计）：期望 `$E(X)$`、方差 `$D(X)$`、
条件概率 `$P(B\mid A)$` 用 `\mid`、分布 `$N(\mu,\sigma^2)$`（第二参数是方差）等。按本课列。

## 5. Markdown / 数学 / 排版硬规则（违反会编译失败或难看；preprocess.py 兜底但仍要主动遵守）
- **数学**：行内 `$...$`，独立 `$$...$$`。**正文不打字面 Unicode 数学符号**（≤ ≥ ≠ → ∞ ∈ ∪ 和希腊字母），一律 LaTeX：`$\le$`、`$\mu$`、`$\sigma^2$`。
- **不要把行内数学包进反引号**（`` `$\le$` `` 会被当字面代码原样输出）——数学示范要写就用纯文字描述「行内用美元号」，别用反引号包。
- **闭合 `$` 后紧跟数字会失效** → 写进数学段 `$\sigma^2$` 或加空格 `$\sigma$ 2`。
- **下标接字体命令要花括号**：`$X_{\max}$`、`$_{\mathbf{x}}$`，不写 `$_\mathbf{x}$`。
- 条件概率竖线用 `\mid`；分段函数用 `cases`；数学环境内中文用 `\text{中文}`；重要结论 `\boxed{...}`。
- 表格用标准 Markdown 管道表格（booktabs 由 pandoc 自动加，**别手动 `\usepackage{booktabs}`**）。
- **callout 四类** `[!note]`/`[!example]`/`[!warning]`/`[!tip]`：渲染器（callout.lua）会**自动给框加上彩色类型标签**（直觉 / 例 / 易错 / 技巧），所以 `[!type]` 后**只写具体标题或留空，切勿再重复类型名**。正确：`> [!warning] mid2024 第3题`（渲成「易错：mid2024 第3题」）或 `> [!warning]`（渲成「易错」）。**错误**：`> [!warning] 易错（mid2024 第3题）`（会渲成「易错：易错（mid2024 第3题）」，类型名重复——曾大面积中招）。标签与标题同行、正文换行、每行 `>` 前缀；克制用（每小节 1–2 个）。
- **禁**：`\usepackage{enumitem}`（basic TeX Live 未装）、`\usepackage{booktabs}`、自定义 `\newcommand`（组装不生效，直接写全宏）、HTML、mermaid（PDF 不渲染）、圈数字 ①②③（用 `(1)(2)`）、希腊字母当列表标签、正文裸 `---/***` 分隔线（会被当 YAML 报错）、★/emoji。列表用 `1./2./-`。
- 关键概念首次出现给中英文。

## 6.〔按课程填〕考情分析
从往年真题归纳：卷子结构（题型+分值）+ 每章对应哪些真题题型。这是 §0「以真题为纲」的依据，务必让每章的写作 agent 内化属于自己那章的部分。

## 7. 质量底线
推导/计算自己 trace 一遍确认对，例题每步数值算对；直觉真讲清，不写「显然/容易看出」；输出讲义正文，不要「我写了……」元叙述；发现真题/作业疑似错用 `> [!warning]` 标注修正理由。

## 8. 输出
整章 Markdown 写入指定文件（首行 `# 第 N 章 …`，无 YAML），然后只回一行：本章标题 + 大致字数/小节数 + 覆盖了哪些真题题型 + 是否发现疑似错误。
